# MeeTeam 인수조건 명세서

> 5W1H 분석 기반 인수조건 문서
> 비개발자도 이해할 수 있는 명확한 시나리오 중심으로 작성

---

## 용어 정의

| 용어 | 설명 |
|------|------|
| 비회원 | 로그인하지 않은 사용자 |
| 회원 | 로그인한 사용자 |
| 프로젝트 리더 | 프로젝트를 생성한 사람, 팀의 관리 권한 보유 |
| 팀원 | 프로젝트에 참여 중인 회원 (리더 제외) |
| 지원자 | 프로젝트에 지원한 회원 (승인 대기 중) |
| 메이커 | MeeTeam에서 활동하는 모든 회원을 지칭 |

## 1. 회원 인증

### 1.1 이메일 회원가입

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 비회원 (MeeTeam을 처음 이용하는 사람) |
| **What** | 이메일과 개인정보를 입력하여 계정을 생성한다 |
| **When** | MeeTeam 서비스를 이용하고 싶을 때 |
| **Where** | 회원가입 페이지 |
| **Why** | 프로젝트 참여, 팀원 찾기 등 서비스 이용을 위해 |
| **How** | 이메일, 비밀번호, 기본정보, 관심분야 등을 입력 후 가입 버튼 클릭 |

#### 인수조건

```gherkin
기능: 이메일 회원가입
  MeeTeam 서비스를 이용하기 위해 비회원이 계정을 생성할 수 있어야 한다.

  시나리오: 모든 필수 정보를 입력하면 회원가입에 성공한다
    먼저 비회원이 회원가입 페이지에 접속한다
    그리고 이메일 "hong@example.com"을 입력한다
    그리고 이메일 중복 확인을 클릭하여 "사용 가능한 이메일입니다" 메시지를 확인한다
    그리고 비밀번호 "password123"을 입력한다
    그리고 비밀번호 확인에 "password123"을 입력한다
    그리고 이름 "홍길동"을 입력한다
    그리고 생년월일 "1995-03-15"를 선택한다
    그리고 성별 "남성"을 선택한다
    그리고 관심 분야 "프론트엔드 - 웹 프론트엔드"를 선택한다
    만약 "가입하기" 버튼을 클릭하면
    그러면 회원가입에 성공한다
    그리고 로그인 페이지로 이동한다

  시나리오: 이미 가입된 이메일로는 가입할 수 없다
    먼저 "hong@example.com" 이메일로 가입된 회원이 존재한다
    그리고 비회원이 회원가입 페이지에 접속한다
    그리고 이메일 "hong@example.com"을 입력한다
    만약 이메일 중복 확인을 클릭하면
    그러면 "이미 존재하는 이메일입니다" 메시지가 표시된다
    그리고 가입하기 버튼이 비활성화된다

  시나리오: 비밀번호가 8자 미만이면 가입할 수 없다
    먼저 비회원이 회원가입 페이지에 접속한다
    그리고 비밀번호 "pass123"을 입력한다
    그러면 "비밀번호는 최소 8자 이상이어야 합니다" 메시지가 표시된다

  시나리오: 비밀번호 확인이 일치하지 않으면 가입할 수 없다
    먼저 비회원이 회원가입 페이지에 접속한다
    그리고 비밀번호 "password123"을 입력한다
    그리고 비밀번호 확인에 "password456"을 입력한다
    그러면 "비밀번호가 일치하지 않습니다" 메시지가 표시된다

  시나리오: 필수 항목을 입력하지 않으면 가입할 수 없다
    먼저 비회원이 회원가입 페이지에 접속한다
    그리고 이메일만 입력하고 다른 필수 항목은 비워둔다
    만약 "가입하기" 버튼을 클릭하면
    그러면 "필수 항목을 모두 입력해주세요" 메시지가 표시된다
```

### 1.2 소셜 회원가입 (Google/GitHub)

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 비회원 (Google 또는 GitHub 계정 보유자) |
| **What** | 소셜 계정으로 빠르게 가입한다 |
| **When** | 이메일/비밀번호 입력 없이 간편하게 가입하고 싶을 때 |
| **Where** | 로그인 페이지 → 소셜 로그인 버튼 → 추가정보 입력 페이지 |
| **Why** | 간편한 가입 절차로 사용자 경험 향상 |
| **How** | 소셜 로그인 버튼 클릭 → 소셜 인증 → 추가 정보 입력 → 가입 완료 |

#### 인수조건

```gherkin
기능: 소셜 회원가입
  Google 또는 GitHub 계정으로 간편하게 가입할 수 있어야 한다.

  시나리오: Google 계정으로 신규 가입한다
    먼저 비회원이 로그인 페이지에 접속한다
    그리고 "Google" 버튼을 클릭한다
    그리고 Google 계정 인증에 성공한다
    그러면 추가 정보 입력 페이지로 이동한다
    그리고 이름, 생년월일, 성별, 관심 분야를 입력한다
    만약 "가입하기" 버튼을 클릭하면
    그러면 회원가입에 성공한다
    그리고 메인 페이지로 이동한다
    그리고 로그인 상태가 유지된다

  시나리오: 이미 가입된 Google 계정으로 로그인한다
    먼저 "hong@gmail.com" Google 계정으로 가입된 회원이 존재한다
    그리고 비회원이 로그인 페이지에 접속한다
    만약 "Google" 버튼을 클릭하고 동일한 계정으로 인증하면
    그러면 추가 정보 입력 없이 바로 로그인된다
    그리고 메인 페이지로 이동한다
```

### 1.3 로그인

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 가입된 회원 |
| **What** | 이메일과 비밀번호로 로그인한다 |
| **When** | MeeTeam 서비스를 이용하고자 할 때 |
| **Where** | 로그인 페이지 |
| **Why** | 회원 전용 기능(프로젝트 등록, 지원 등)을 이용하기 위해 |
| **How** | 이메일, 비밀번호 입력 후 로그인 버튼 클릭 |

#### 인수조건

```gherkin
기능: 이메일 로그인
  가입된 회원이 이메일과 비밀번호로 로그인할 수 있어야 한다.

  배경:
    먼저 다음 정보로 가입된 회원이 존재한다:
      | 이메일            | 비밀번호      | 이름    |
      | hong@example.com | password123  | 홍길동  |

  시나리오: 올바른 정보로 로그인에 성공한다
    먼저 비회원이 로그인 페이지에 접속한다
    그리고 이메일 "hong@example.com"을 입력한다
    그리고 비밀번호 "password123"을 입력한다
    만약 "로그인" 버튼을 클릭하면
    그러면 로그인에 성공한다
    그리고 메인 페이지로 이동한다
    그리고 헤더에 "홍길동"의 프로필이 표시된다

  시나리오: 잘못된 비밀번호로 로그인에 실패한다
    먼저 비회원이 로그인 페이지에 접속한다
    그리고 이메일 "hong@example.com"을 입력한다
    그리고 비밀번호 "wrongpassword"를 입력한다
    만약 "로그인" 버튼을 클릭하면
    그러면 "이메일 또는 비밀번호가 올바르지 않습니다" 메시지가 표시된다
    그리고 로그인 페이지에 머무른다

  시나리오: 가입되지 않은 이메일로 로그인에 실패한다
    먼저 비회원이 로그인 페이지에 접속한다
    그리고 이메일 "unknown@example.com"을 입력한다
    그리고 비밀번호 "anypassword"를 입력한다
    만약 "로그인" 버튼을 클릭하면
    그러면 "이메일 또는 비밀번호가 올바르지 않습니다" 메시지가 표시된다
```

### 1.4 로그아웃

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 로그인한 회원 |
| **What** | 로그아웃하여 세션을 종료한다 |
| **When** | 서비스 이용을 마치거나 다른 계정으로 전환하고 싶을 때 |
| **Where** | 헤더의 사용자 메뉴 |
| **Why** | 보안 유지 및 계정 전환을 위해 |
| **How** | 프로필 클릭 → 드롭다운 메뉴에서 로그아웃 클릭 |

#### 인수조건

```gherkin
기능: 로그아웃
  로그인한 회원이 안전하게 로그아웃할 수 있어야 한다.

  시나리오: 로그아웃하면 비회원 상태가 된다
    먼저 "홍길동" 회원이 로그인한 상태이다
    만약 헤더의 프로필을 클릭하고 "로그아웃"을 선택하면
    그러면 로그아웃에 성공한다
    그리고 메인 페이지로 이동한다
    그리고 헤더에 "로그인" 버튼이 표시된다

  시나리오: 로그아웃 후 이전 토큰으로 접근할 수 없다
    먼저 "홍길동" 회원이 로그인 후 로그아웃한 상태이다
    만약 이전에 발급받은 토큰으로 회원 전용 기능에 접근하면
    그러면 "로그인이 필요합니다" 메시지가 표시된다
    그리고 로그인 페이지로 이동한다
```

## 2. 프로젝트

### 2.1 프로젝트 등록

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 로그인한 회원 (프로젝트를 시작하고 싶은 사람) |
| **What** | 새로운 프로젝트를 등록하고 팀원을 모집한다 |
| **When** | 사이드 프로젝트를 시작하고 함께할 팀원이 필요할 때 |
| **Where** | 프로젝트 등록 페이지 |
| **Why** | 아이디어를 실현할 팀을 구성하기 위해 |
| **How** | 프로젝트 정보 입력 → 모집 포지션 설정 → 등록 버튼 클릭 |

#### 인수조건

```gherkin
기능: 프로젝트 등록
  회원이 새로운 프로젝트를 등록하고 팀원을 모집할 수 있어야 한다.

  배경:
    먼저 "홍길동" 회원이 로그인한 상태이다

  시나리오: 모든 정보를 입력하여 프로젝트를 등록한다
    먼저 "프로젝트 등록하기" 페이지에 접속한다
    그리고 프로젝트명 "AI 기반 뉴스 요약 서비스"를 입력한다
    그리고 프로젝트 카테고리 "AI/테크"를 선택한다
    그리고 프로젝트 소개글을 입력한다
    그리고 출시 플랫폼 "Web"을 선택한다
    그리고 나의 분야 "백엔드 - 웹 서버"를 선택한다
    그리고 모집 분야를 추가한다:
      | 분야       | 상세         | 인원 |
      | 프론트엔드 | React Native | 2명  |
      | 디자인     | UI/UX        | 1명  |
    그리고 필요 기술 스택 "React, TypeScript"를 입력한다
    그리고 모집 마감일 "2024-12-31"을 선택한다
    만약 "프로젝트 등록하기" 버튼을 클릭하면
    그러면 프로젝트 등록에 성공한다
    그리고 생성된 프로젝트 상세 페이지로 이동한다
    그리고 "홍길동"이 프로젝트 리더로 표시된다

  시나리오: 프로젝트명 없이 등록할 수 없다
    먼저 "프로젝트 등록하기" 페이지에 접속한다
    그리고 프로젝트명을 비워둔다
    그리고 다른 필수 정보를 모두 입력한다
    만약 "프로젝트 등록하기" 버튼을 클릭하면
    그러면 "프로젝트명을 입력해주세요" 메시지가 표시된다

  시나리오: 모집 분야 없이 등록할 수 없다
    먼저 "프로젝트 등록하기" 페이지에 접속한다
    그리고 프로젝트 기본 정보를 모두 입력한다
    그리고 모집 분야를 추가하지 않는다
    만약 "프로젝트 등록하기" 버튼을 클릭하면
    그러면 "최소 1개 이상의 모집 분야를 추가해주세요" 메시지가 표시된다

  시나리오: 비회원은 프로젝트를 등록할 수 없다
    먼저 비회원이 "프로젝트 등록하기" 페이지에 접속을 시도한다
    그러면 로그인 페이지로 이동한다
    그리고 "로그인이 필요한 서비스입니다" 메시지가 표시된다
```

### 2.2 프로젝트 검색

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 모든 사용자 (비회원 포함) |
| **What** | 원하는 조건의 프로젝트를 검색하고 필터링한다 |
| **When** | 참여하고 싶은 프로젝트를 찾고 싶을 때 |
| **Where** | 프로젝트 찾기 페이지 |
| **Why** | 자신에게 맞는 프로젝트를 빠르게 찾기 위해 |
| **How** | 검색어 입력 또는 필터(카테고리, 플랫폼, 분야) 선택 |

#### 인수조건

```gherkin
기능: 프로젝트 검색
  사용자가 원하는 조건으로 프로젝트를 검색할 수 있어야 한다.

  배경:
    먼저 다음 프로젝트들이 등록되어 있다:
      | 프로젝트명           | 카테고리   | 플랫폼 | 모집상태  |
      | AI 뉴스 요약 서비스  | AI/테크   | Web   | 모집중   |
      | 헬스케어 앱         | 헬스케어   | iOS   | 모집중   |
      | 여행 동행 플랫폼     | 여행      | Web   | 모집완료  |
      | 반려동물 케어 앱     | 반려동물   | Android | 모집중 |

  시나리오: 프로젝트명으로 검색한다
    먼저 사용자가 "프로젝트 찾기" 페이지에 접속한다
    만약 검색창에 "뉴스"를 입력하고 검색하면
    그러면 "AI 뉴스 요약 서비스" 프로젝트가 표시된다
    그리고 다른 프로젝트는 표시되지 않는다

  시나리오: 카테고리로 필터링한다
    먼저 사용자가 "프로젝트 찾기" 페이지에 접속한다
    만약 카테고리 "AI/테크"를 선택하면
    그러면 "AI 뉴스 요약 서비스" 프로젝트만 표시된다

  시나리오: 플랫폼으로 필터링한다
    먼저 사용자가 "프로젝트 찾기" 페이지에 접속한다
    만약 플랫폼 "iOS"를 선택하면
    그러면 "헬스케어 앱" 프로젝트만 표시된다

    그리고 모집 중인 3개의 프로젝트만 표시된다

  시나리오: 검색 결과가 없으면 안내 메시지가 표시된다
    먼저 사용자가 "프로젝트 찾기" 페이지에 접속한다
```

## 3. 프로젝트 지원

### 3.1 지원하기

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 로그인한 회원 (프로젝트에 참여하고 싶은 사람) |
| **What** | 프로젝트의 특정 포지션에 지원한다 |
| **When** | 마음에 드는 프로젝트를 찾았을 때 |
| **Where** | 프로젝트 상세 페이지 → 지원하기 모달 |
| **Why** | 프로젝트 팀에 합류하기 위해 |
| **How** | 지원하기 버튼 클릭 → 지원 사유 작성 → 지원 제출 |

#### 인수조건

```gherkin
기능: 프로젝트 지원
  회원이 프로젝트에 지원하여 팀에 합류를 요청할 수 있어야 한다.

  배경:
    먼저 "홍길동"이 리더인 "AI 뉴스 요약 서비스" 프로젝트가 존재한다
    그리고 프론트엔드 포지션 2명을 모집 중이다
    그리고 "김철수" 회원이 로그인한 상태이다

  시나리오: 프로젝트에 지원한다
    먼저 "김철수"가 프로젝트 상세 페이지에서 "팀원 모집" 탭을 클릭한다
    그리고 프론트엔드 포지션의 "지원하기" 버튼을 클릭한다
    그러면 지원하기 모달이 표시된다
    그리고 "김철수"의 프로필 정보가 자동으로 입력되어 있다
    그리고 지원 사유를 작성한다:
      """
      리액트와 타입스크립트를 주로 사용하며 사이드 프로젝트 경험을 쌓고 싶어 지원했습니다.
      """
    만약 "지원하기" 버튼을 클릭하면
    그러면 "지원이 완료되었습니다" 메시지가 표시된다
    그리고 "홍길동" 리더에게 새로운 지원 알림이 발송된다

  시나리오: 이미 지원한 프로젝트에 중복 지원할 수 없다
    먼저 "김철수"가 이미 프로젝트에 지원한 상태이다
    만약 같은 포지션의 "지원하기" 버튼을 클릭하면
    그러면 "이미 지원한 프로젝트입니다" 메시지가 표시된다

  시나리오: 이미 참여 중인 프로젝트에 지원할 수 없다
    먼저 "김철수"가 이미 프로젝트의 팀원인 상태이다
    만약 다른 포지션의 "지원하기" 버튼을 클릭하면
    그러면 "이미 참여 중인 프로젝트입니다" 메시지가 표시된다

  시나리오: 모집 마감된 프로젝트에 지원할 수 없다
    먼저 프로젝트의 모집 상태가 "모집완료"인 상태이다
    만약 "김철수"가 프로젝트 상세 페이지에 접속하면
    그러면 "지원하기" 버튼이 표시되지 않는다
    그리고 "모집이 마감되었습니다" 메시지가 표시된다

  시나리오: 비회원은 지원할 수 없다
    먼저 비회원이 프로젝트 상세 페이지에 접속한다
    만약 "지원하기" 버튼을 클릭하면
    그러면 로그인 페이지로 이동한다
```

```gherkin
기능: 지원자 관리

    먼저 "홍길동"이 리더인 프로젝트가 존재한다
    그리고 프론트엔드 포지션 모집 정원은 2명이다
    그리고 "김철수"와 "이영희"가 지원한 상태이다
    그리고 "홍길동"이 로그인한 상태이다

  시나리오: 지원자 목록을 확인한다
    먼저 "홍길동"이 프로젝트 관리 페이지의 "지원자 관리" 탭에 접속한다
    그러면 "지원자 목록 2"가 표시된다
    그리고 각 지원자의 정보가 표시된다:
      | 이름    | 지원분야    | 지원일      |
      | 김철수  | 프론트엔드  | 2024-03-10 |
      | 이영희  | 프론트엔드  | 2024-03-11 |

  시나리오: 지원을 승인하면 팀원으로 등록된다
    먼저 "홍길동"이 지원자 관리 페이지에서 "김철수"의 지원서를 확인한다
    만약 "승인" 버튼을 클릭하면
    그러면 "김철수님의 지원이 승인되었습니다" 메시지가 표시된다
    그리고 "김철수"가 프로젝트 팀원 목록에 추가된다
    그리고 "김철수"에게 "지원이 승인되었습니다" 알림이 발송된다
    그리고 프론트엔드 포지션 현황이 "1/2명"으로 업데이트된다

  시나리오: 지원을 거절한다
    먼저 "홍길동"이 지원자 관리 페이지에서 "이영희"의 지원서를 확인한다
    만약 "거절" 버튼을 클릭하면
    그러면 "이영희님의 지원이 거절되었습니다" 메시지가 표시된다
    그리고 "이영희"에게 "지원이 거절되었습니다" 알림이 발송된다
    그리고 지원자 목록에서 "이영희"가 제거된다

  시나리오: 모집 정원이 가득 차면 더 이상 승인할 수 없다
    먼저 프론트엔드 포지션에 이미 2명이 승인된 상태이다
    그리고 "박지민"이 추가로 지원한 상태이다
    만약 "홍길동"이 "박지민"의 지원을 승인하려고 하면
    그러면 "모집 정원이 가득 찼습니다" 메시지가 표시된다
    그리고 승인이 거부된다
```

### 4.3 프로젝트 상태 관리

```gherkin
기능: 프로젝트 상태 관리

    그리고 "홍길동"이 프로젝트 관리 페이지에 접속한다
    만약 모집 상태를 "모집완료"로 변경하면
    그러면 "모집 상태가 변경되었습니다" 메시지가 표시된다
    그리고 프로젝트 상세 페이지에서 "지원하기" 버튼이 사라진다

  시나리오: 프로젝트 상태를 "완료됨"으로 변경한다
    먼저 "홍길동"이 리더인 프로젝트의 상태가 "진행중"이다
    그리고 "홍길동"이 프로젝트 관리 페이지에 접속한다
    만약 프로젝트 상태를 "완료됨"으로 변경하면
    그러면 "프로젝트 상태가 변경되었습니다" 메시지가 표시된다
    그리고 프로젝트 카드에 "완료됨" 태그가 표시된다
```

## 5. 팀원 찾기

### 5.1 팀원 검색

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 모든 사용자 (프로젝트 리더 또는 팀원을 찾는 사람) |
| **What** | 원하는 조건의 팀원을 검색한다 |
| **When** | 프로젝트에 필요한 인재를 찾고 싶을 때 |
| **Where** | 팀원 찾기 페이지 |
| **Why** | 적합한 팀원을 효율적으로 찾기 위해 |
| **How** | 검색어 입력 또는 필터(분야, 기술스택) 선택 |

#### 인수조건

```gherkin
기능: 팀원 찾기
  사용자가 원하는 조건으로 팀원을 검색할 수 있어야 한다.

  배경:
    먼저 다음 회원들이 등록되어 있다:
      | 이름    | 분야       | 기술스택           | 프로젝트경험 |
      | 김철수  | 프론트엔드 | React, Next.js    | 3회        |
      | 이영희  | 백엔드     | Spring, Java      | 5회        |
      | 박지민  | 디자이너   | Figma, Sketch     | 2회        |

  시나리오: 분야로 필터링한다
    먼저 사용자가 "팀원 찾기" 페이지에 접속한다
    만약 분야 "프론트엔드"를 선택하면
    그러면 "김철수"만 검색 결과에 표시된다

  시나리오: 기술 스택으로 필터링한다
    먼저 사용자가 "팀원 찾기" 페이지에 접속한다
    만약 기술 스택 "React"를 추가하면
    그러면 "김철수"만 검색 결과에 표시된다

  시나리오: 프로젝트 경험 많은 순으로 정렬한다
    먼저 사용자가 "팀원 찾기" 페이지에 접속한다
    만약 정렬 "프로젝트 경험 많은 순"을 선택하면
    그러면 "이영희"(5회)가 첫 번째로 표시된다
    그리고 "김철수"(3회)가 두 번째로 표시된다
    그리고 "박지민"(2회)가 세 번째로 표시된다

  시나리오: 이름으로 검색한다
    먼저 사용자가 "팀원 찾기" 페이지에 접속한다
    만약 검색창에 "김철"을 입력하면
    그러면 "김철수"가 검색 결과에 표시된다
```

## 6. 사용자 프로필

### 6.1 프로필 조회

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 모든 사용자 |
| **What** | 다른 회원의 프로필을 조회한다 |
| **When** | 팀원 후보나 지원자에 대해 더 알고 싶을 때 |
| **Where** | 사용자 프로필 페이지 |
| **Why** | 협업 가능성을 판단하기 위해 |
| **How** | 프로필 카드 또는 이름 클릭 |

#### 인수조건

```gherkin
기능: 프로필 조회
  사용자가 다른 회원의 프로필을 확인할 수 있어야 한다.

  배경:
    먼저 "김철수" 회원이 다음 정보로 등록되어 있다:
      | 항목         | 값                           |
      | 이름         | 김철수                        |
      | 직군         | 프론트엔드 - 웹 프론트엔드     |
      | 나이         | 28세                         |
      | 기술스택     | React, TypeScript, Next.js   |
      | 프로젝트경험 | 3회                          |
      | GitHub      | github.com/kimcs            |

  시나리오: 다른 회원의 프로필을 조회한다
    먼저 사용자가 "김철수"의 프로필 페이지에 접속한다
    그러면 "김철수"의 이름이 표시된다
    그리고 직군 "프론트엔드"가 표시된다
    그리고 나이 "28세"가 표시된다
    그리고 기술 스택 "React, TypeScript, Next.js"가 표시된다
    그리고 프로젝트 경험 "3회"가 표시된다
    그리고 GitHub 링크가 표시된다

  시나리오: 로그인한 회원은 제안 보내기 버튼이 표시된다
    먼저 "홍길동"이 로그인한 상태이다
    만약 "김철수"의 프로필 페이지에 접속하면
    그러면 "제안 보내기" 버튼이 표시된다

  시나리오: 자신의 프로필에는 프로필 수정 버튼이 표시된다
    먼저 "김철수"가 로그인한 상태이다
    만약 자신의 프로필 페이지에 접속하면
    그러면 "프로필 수정" 버튼이 표시된다
    그리고 "제안 보내기" 버튼은 표시되지 않는다
```

### 6.2 프로필 수정

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 로그인한 회원 (자신의 프로필을 수정하려는 사람) |
| **What** | 자신의 프로필 정보를 수정한다 |
| **When** | 정보가 변경되었거나 프로필을 업데이트하고 싶을 때 |
| **Where** | 마이페이지 → 프로필 수정 페이지 |
| **Why** | 최신 정보를 반영하여 더 나은 매칭을 위해 |
| **How** | 프로필 수정 버튼 클릭 → 정보 수정 → 저장 |

#### 인수조건

```gherkin
기능: 프로필 수정
  회원이 자신의 프로필 정보를 수정할 수 있어야 한다.

  배경:
    먼저 "김철수" 회원이 로그인한 상태이다

  시나리오: 기본 정보를 수정한다
    먼저 "김철수"가 프로필 수정 페이지에 접속한다
    그리고 직군을 "백엔드 - 웹 서버"로 변경한다
    만약 "저장하기" 버튼을 클릭하면
    그러면 "프로필이 수정되었습니다" 메시지가 표시된다
    그리고 프로필에 "백엔드"가 표시된다

  시나리오: 기술 스택을 추가한다
    먼저 "김철수"가 프로필 수정 페이지에 접속한다
    그리고 기술 스택에 "Vue.js"를 추가한다
    만약 "저장하기" 버튼을 클릭하면
    그러면 프로필에 "Vue.js"가 추가로 표시된다

  시나리오: 자기소개를 작성한다
    먼저 "김철수"가 프로필 수정 페이지에 접속한다
    그리고 자기소개란에 "3년차 프론트엔드 개발자입니다"를 입력한다
    만약 "저장하기" 버튼을 클릭하면
    그러면 프로필에 자기소개가 표시된다

  시나리오: 프로필 사진을 변경한다
    먼저 "김철수"가 프로필 수정 페이지에 접속한다
    그리고 프로필 사진 영역의 카메라 아이콘을 클릭한다
    그리고 새로운 이미지 파일을 업로드한다
    만약 "저장하기" 버튼을 클릭하면
    그러면 프로필 사진이 변경된다
```

## 7. Q&A

### 7.1 질문/답변

#### 5W1H 분석
| 항목 | 내용 |
|------|------|
| **Who** | 회원 (프로젝트에 관심 있는 사람) / 프로젝트 리더 |
| **What** | 프로젝트에 대해 질문하고 답변한다 |
| **When** | 프로젝트 정보가 불충분하거나 궁금한 점이 있을 때 |
| **Where** | 프로젝트 상세 페이지 → Q&A 탭 |
| **Why** | 지원 전 궁금한 점을 해소하기 위해 |
| **How** | 질문 작성 → 리더가 답변 작성 |

#### 인수조건

```gherkin
기능: 프로젝트 Q&A
  회원이 프로젝트에 대해 질문하고 리더가 답변할 수 있어야 한다.

  배경:
    먼저 "홍길동"이 리더인 "AI 뉴스 요약 서비스" 프로젝트가 존재한다

  시나리오: 프로젝트에 질문을 작성한다
    먼저 "김철수"가 로그인한 상태이다
    그리고 프로젝트 상세 페이지의 "Q&A" 탭에 접속한다
    그리고 질문 입력란에 "프로젝트 모임은 매주 무슨 요일에 진행하나요?"를 입력한다
    만약 "등록" 버튼을 클릭하면
    그러면 질문이 Q&A 목록에 추가된다
    그리고 "홍길동" 리더에게 새로운 질문 알림이 발송된다

  시나리오: 리더가 질문에 답변한다
    먼저 "홍길동"이 로그인한 상태이다
    그리고 "김철수"가 작성한 질문이 존재한다
    만약 "홍길동"이 해당 질문에 "매주 토요일 오후 2시 강남역 부근에서 진행합니다"라고 답변하면
    그러면 답변이 질문 아래에 표시된다
    그리고 답변 옆에 "팀장" 태그가 표시된다
    그리고 "김철수"에게 답변 알림이 발송된다

  시나리오: 비회원은 질문을 작성할 수 없다
    먼저 비회원이 프로젝트 Q&A 탭에 접속한다
    그러면 질문 입력란이 표시되지 않는다
    그리고 "질문을 작성하려면 로그인이 필요합니다" 메시지가 표시된다
```
