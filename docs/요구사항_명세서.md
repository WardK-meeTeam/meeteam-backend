# MeeTeam 요구사항 명세서

> UI 스크린샷 기반 요구사항 분석 문서

---

## 1. 회원 (Member)

### 1.1 로그인
| 요구사항 ID | 요구사항 | 상세 설명 |
|------------|---------|----------|
| AUTH-001 | 이메일 로그인 | 이메일과 비밀번호로 로그인 |
| AUTH-002 | Google 소셜 로그인 | Google OAuth를 통한 로그인 |
| AUTH-003 | GitHub 소셜 로그인 | GitHub OAuth를 통한 로그인 |
| AUTH-004 | 로그아웃 | 로그아웃 시 토큰 무효화 |
| AUTH-005 | 토큰 재발급 | Access Token 만료 시 Refresh Token으로 재발급 |

### 1.2 회원가입 (이메일)
| 요구사항 ID | 요구사항 | 상세 설명 |
|------------|---------|----------|
| SIGNUP-001 | 이메일 입력 | 이메일 형식 검증 |
| SIGNUP-002 | 이메일 중복 확인 | "중복 확인" 버튼으로 이메일 중복 체크 |
| SIGNUP-003 | 비밀번호 입력 | 8자 이상 입력 |
| SIGNUP-004 | 비밀번호 확인 | 비밀번호 재입력으로 일치 여부 확인 |
| SIGNUP-005 | 이름 입력 | 실명 입력 (필수) |
| SIGNUP-006 | 생년월일 입력 | 연도-월-일 형식 (필수) |
| SIGNUP-007 | 성별 선택 | 남성/여성 선택 (필수) |
| SIGNUP-008 | 관심 분야 선택 | 프론트엔드, 상세 분야 (예: UI/UX) 선택, 분야 추가 가능 |
| SIGNUP-009 | 기술 스택 입력 | 기술 스택 입력 후 Enter (예: React, Figma) - 선택 |
| SIGNUP-010 | 프로젝트 경험 횟수 | 숫자 입력 (회) |
| SIGNUP-011 | GitHub URL 입력 | github.com/... 형식 - 선택 |
| SIGNUP-012 | 블로그 URL 입력 | URL 형식 - 선택 |
| SIGNUP-013 | 프로필 사진 업로드 | JPG, PNG (최대 10MB) |
| SIGNUP-014 | 이용약관 동의 | 이용약관 및 개인정보처리방침 동의 필수 |

### 1.3 회원가입 (소셜 로그인)
| 요구사항 ID | 요구사항 | 상세 설명 |
|------------|---------|----------|
| OAUTH-SIGNUP-001 | 추가 정보 입력 | OAuth 인증 후 추가 정보 입력 페이지로 이동 |
| OAUTH-SIGNUP-002 | 입력 항목 | 이름, 생년월일, 성별, 관심분야, 기술스택, 프로젝트 경험 횟수, GitHub, 블로그, 프로필 사진 |

| PROJECT-009 | 모집 분야 | 직군 선택, 상세 분야 선택, 모집 인원 (+/- 버튼), 추가하기 버튼 |
| PROJECT-010 | 필요 기술 스택 | 프론트엔드/백엔드 등 분야별 기술 입력 후 Enter |

## 10. 데이터 모델 예상

### 10.1 Member (회원)
```
- id: Long
- email: String
- password: String (nullable for OAuth)
- name: String
- birthDate: LocalDate
- gender: Enum (MALE, FEMALE)
- profileImageUrl: String
- introduction: String (자기소개)
- projectExperienceCount: Integer
- githubUrl: String
- blogUrl: String
- isAvailableForProject: Boolean (프로젝트 참여 여부)
- role: List<MemberRole> (관심 분야)
- skills: List<Skill> (기술 스택)
- oauthProvider: Enum (GOOGLE, GITHUB, null)
- oauthId: String
```

### 10.2 Project (프로젝트)
```
- id: Long
- title: String
- description: String
- coverImageUrl: String
- category: Enum
- platforms: List<Enum> (iOS, Android, Web)
- githubUrl: String
- communicationChannelUrl: String
- recruitmentDeadline: LocalDate (nullable)
- isRecruitmentOpen: Boolean
- status: Enum (IN_PROGRESS, COMPLETED)
- leader: Member
- likeCount: Integer
```

### 10.3 RecruitmentPosition (모집 포지션)
```
- id: Long
- project: Project
- roleCategory: Enum (프론트엔드, 백엔드, 디자인, 기획, 마케팅, 기타)
- roleDetail: String (상세 분야)
- requiredCount: Integer
- currentCount: Integer
- requiredSkills: List<Skill>
```

### 10.4 Applicant (지원)
```
- id: Long
- project: Project
- position: RecruitmentPosition
- member: Member
- introduction: String (지원 사유 및 자기소개)
- status: Enum (PENDING, APPROVED, REJECTED)
- appliedAt: LocalDateTime
```

### 10.5 ProjectMember (프로젝트 멤버)
```
- id: Long
- project: Project
- member: Member
- role: String
- isLeader: Boolean
- joinedAt: LocalDateTime
```

### 10.6 ProjectQnA (Q&A)
```
- id: Long
- project: Project
- author: Member
- question: String
- answer: String
- createdAt: LocalDateTime
- answeredAt: LocalDateTime
```
