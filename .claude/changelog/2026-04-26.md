## 5. Refresh Token API HttpOnly 쿠키 갱신 수정 ✅

### 배경
- 프론트엔드가 HttpOnly 쿠키 기반 인증 사용
- 기존 `/api/v1/auth/refresh`는 body로만 access token 반환 (쿠키 갱신 안 함)
- 프론트에서 body 값을 저장할 수 없어 로그인 유지 불가

### 변경 파일

#### AuthService.java
```java
// 기존: String refreshAccessToken(request) - access token만 반환
// 변경: TokenExchangeResult refreshTokens(request) - Token Rotation 적용

public TokenExchangeResult refreshTokens(HttpServletRequest request) {
    // refresh token 검증 후
    String newAccessToken = jwtUtil.createAccessToken(member);
    String newRefreshToken = jwtUtil.createRefreshToken(member);
    return new TokenExchangeResult(newAccessToken, newRefreshToken);
}
```

#### AuthController.java
```java
@PostMapping("/refresh")
public SuccessResponse<String> refreshToken(HttpServletRequest request, HttpServletResponse response) {
    TokenExchangeResult result = authService.refreshTokens(request);

    // 새로운 Access Token 쿠키 설정
    accessTokenCookieProvider.addCookie(response, result.getAccessToken());
    // 새로운 Refresh Token 쿠키 설정 (Token Rotation)
    refreshTokenCookieProvider.addCookie(response, result.getRefreshToken());

    return SuccessResponse.onSuccess("토큰이 갱신되었습니다.");
}
```

### 프론트엔드 사용법
```javascript

  method: 'POST',
  credentials: 'include'  // 쿠키 전송
});

if (response.ok) {
  // 브라우저가 자동으로 Set-Cookie 처리
  // accessToken, refreshToken 쿠키 갱신됨
  // 실패했던 API 재요청
}
```

## 6. 지원자 상세보기 API age null 버그 수정 ✅

### 배경
- 지원자 상세보기 API (`GET /api/v1/projects/{projectId}/applications/{applicationId}`)에서 age가 null로 반환됨
- 회원가입 시 `birth`만 설정하고 `age` 필드는 설정하지 않아 발생

### 원인
- `Member` 엔티티에 `age` 필드와 `birth` 필드가 별도로 존재
- 회원가입 팩토리 메서드들(`createMember`, `createOAuthMember`, `createSejongMember`)에서 `birth`만 설정하고 `age`는 설정하지 않음
- Lombok의 `@Getter`가 생성한 `getAge()`는 null인 `age` 필드를 그대로 반환

### 변경 파일

#### Member.java
```java
// 추가된 메서드 - Lombok getter를 오버라이드
public Integer getAge() {
    if (this.age != null) {
        return this.age;
    }
    if (this.birth == null) {
        return null;
    }
    return Period.between(this.birth, LocalDate.now()).getYears();
}
```

### 동작 방식
1. `age` 필드가 설정되어 있으면 해당 값 반환 (프로필 수정으로 직접 설정된 경우)
2. `age`가 null이면 `birth`에서 나이 계산
3. `birth`도 null이면 null 반환
